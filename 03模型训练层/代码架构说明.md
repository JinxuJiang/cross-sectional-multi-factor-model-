## 二、模块职责说明

### 2.1 主入口：`main_train_v1.py`

**职责**：程序的启动入口，负责环境准备和参数解析。

**主要流程**：
```python
1. parse_args()          # 解析命令行参数 (--exp-id, --config, --start-date 等)
2. load_config()         # 加载 YAML 配置文件
3. check_data_files()    # 检查必要数据文件是否存在
4. WalkForwardTrainerV1(config, exp_id)  # 初始化训练器
5. trainer.run()         # 执行训练
```

**关键参数**：
- `--exp-id`：实验ID，决定输出目录名称
- `--config`：配置文件路径（如 `configs/rank_config.yaml`）
- `--gap`：训练-验证/验证-测试之间的隔离天数
- `--horizon`：标签预测周期（默认20天）

---

### 2.2 数据切分器：`splitter_v1.py`

**类**：`WalkForwardSplitterV1`

**职责**：预计算所有 Fold 的时间范围，确保训练/验证/测试集之间有 Gap 隔离。

**核心逻辑**：
```python
# 初始化时一次性计算所有 Fold
self._splits = self._compute_splits()

# 结果格式：[(train_dates, valid_dates, test_dates), ...] 共51个元组
```

**时间窗口配置**（来自 YAML）：
| 参数 | 含义 | 日历天 | 估算交易日 |
|------|------|--------|-----------|
| `train_window='3Y'` | 训练集长度 | 1095天 | 756天 (×0.69) |
| `valid_window='6M'` | 验证集长度 | 180天 | 126天 (×0.69) |
| `test_window='3M'` | 测试集长度 | 90天 | 63天 (×0.69) |
| `step='3M'` | 滚动步长 | 90天 | 63天 (×0.69) |

**双重 Gap 机制**：
- `gap_train_valid=21`：训练集和验证集之间间隔21个交易日
- `gap_valid_test=21`：验证集和测试集之间间隔21个交易日

**为什么需要 Gap？**
```
标签计算使用 T+1开盘买入 → T+21开盘卖出
训练集最后一天用到 T+21 的价格
验证集必须从 T+22 开始，否则训练集标签会"看到"验证集的未来信息
```

### 2.3 数据构造器：`data_constructor_v1.py`

**类**：`DataConstructorV1`

**职责**：根据日期列表构造 X（特征矩阵）和 y（标签向量）。

**核心方法**：

#### `build(dates)` - 构造训练数据
```python
# 输入：日期列表（如 Fold 0 的训练日期）
# 输出：
X: DataFrame (n_samples, n_features), MultiIndex=(date, stock_code)
y: Series (n_samples,), MultiIndex=(date, stock_code)
```

**内部流程**：
```
1. _discover_factor_files()   # 扫描因子目录，发现所有因子文件
2. _load_factor_data()        # 按需加载因子数据（惰性加载）
3. _load_close_data()         # 加载收盘价（用于确定股票池）
4. _load_open_data()          # 加载开盘价（V1新增，用于计算标签）
5. _compute_labels()          # 计算未来收益率
6. _construct_cross_section() # 截面数据对齐（剔除缺失值）
```

**标签计算逻辑（V1修复版）**：
```python
if use_open_price:  # 默认 True
    # T日收盘后计算因子
    # T+1日开盘买入 (open[t+1])
    # T+21日开盘卖出 (open[t+21])
    label = open[t+21] / open[t+1] - 1
else:
    # 旧版（有执行时点偏差）
    label = close[t+20] / close[t] - 1
```

**为什么使用 MultiIndex？**
- 便于截面操作：`df.groupby(level=0).rank()` 按日期分组排名
- LambdaRank 模型需要按日期切分 group 信息

### 2.4 训练器：`walk_forward_trainer_v1.py`

**类**：`WalkForwardTrainerV1`

**职责**：协调整个训练流程，是系统的"指挥中心"。

**核心方法调用链**：

```python
run()  # 主入口
    ├── _save_config()                    # 保存实验配置
    ├── _initialize_splitter()            # 初始化 splitter（获取交易日列表）
    │       └── data_constructor._load_close_data()  # 第一次真正读文件
    │       └── WalkForwardSplitterV1(...) # 预计算所有 Fold
    │
    ├── for fold_id, dates in splitter:   # 遍历所有 Fold
    │       └── _train_fold(fold_id, train_dates, valid_dates, test_dates)
    │               ├── data_constructor.build(train_dates)  # 构造训练数据
    │               ├── data_constructor.build(valid_dates)  # 构造验证数据
    │               ├── data_constructor.build(test_dates)   # 构造测试数据
    │               ├── LightGBMModel(config)                # 初始化模型
    │               ├── model.fit(X_train, y_train, X_valid, y_valid)  # 训练
    │               ├── model.predict(X_test)                # 预测
    │               ├── model.save(path)                     # 保存模型
    │               └── model.get_feature_importance()       # 获取特征重要性
    │
    ├── _aggregate_predictions()          # 合并所有 Fold 预测结果
    │       ├── pd.concat(all_predictions)
    │       ├── drop_duplicates()         # 去重（保留 fold_id 大的）
    │       └── to_parquet(predictions.parquet)
    │
    ├── _generate_summary_report()        # 生成汇总报告
    │       ├── 计算每个 Fold 的 IC、Rank IC
    │       └── to_parquet(summary.parquet)
    │
    ├── _generate_visualizations()        # 生成可视化图表
    │       ├── _plot_ic_trend()          # IC 趋势图 → ic_trend_v1.png
    │       └── _plot_feature_importance() # 特征重要性图 → feature_importance_v1.png
    │
    └── _generate_live_predictions()      # 生成实盘预测
            ├── 加载最后一个 Fold 的模型
            ├── data_constructor.build_for_prediction(live_dates)
            └── to_parquet(live_predictions.parquet)
```

**状态记录**：
```python
self.all_predictions = []      # 存储每个 Fold 的预测结果
self.last_fold_id = 0          # 记录最后一个 Fold 编号
self.last_model_path = None    # 记录最后一个模型文件路径（用于实盘预测）
```

### 2.5 模型层：`models/`

#### 基类：`base_model.py`

```python
class BaseModel(ABC):

    @classmethod
    def load(path):  # 通用加载
```

#### 回归模型：`lightgbm_model.py`
```python

class LightGBMRankModel(BaseModel):
```

## 三、数据流详解

### 3.1 交易日列表来源
```

    ↓ pq.read_table()

    ↓ index.tolist()
List[Timestamp] (dates)
    ↓ 传给 WalkForwardSplitterV1

```

### 3.2 单个 Fold 的数据流
```

    ↓ data_constructor.build()
加载因子 parquet → 计算标签 → 截面对齐

    ↓ model.fit()
训练好的模型
    ↓ model.predict(X_test)
predictions: ndarray (20万行,)
    ↓
result_df (列式 DataFrame):
    columns: [date, stock_code, pred_score, actual_return, fold_id]
```

### 3.3 全量预测结果合并
```
Fold 0 predictions ──┐
Fold 1 predictions ──┼──► pd.concat() ──► drop_duplicates() ──► predictions.parquet
Fold 2 predictions ──┤      (1000万行)        (去重后900万行)
...                  │
Fold 50 predictions ─┘
```

## 五、关键设计要点

### 5.1 为什么需要双重 Gap？
```
无 Gap 情况（数据泄露）：
训练集最后一天: 2024-01-31
验证集第一天:   2024-02-01
问题：训练集标签需要 T+21 的数据，即用到 2024-02-21 的价格
      但验证集从 2024-02-01 开始，训练集"看到"了验证集的未来！

有 Gap 情况（安全）：
训练集最后一天: 2024-01-31
Gap 21天:       2024-02-01 ~ 2024-02-28
验证集第一天:   2024-02-29
安全：训练集标签用到 2024-02-21，早于验证集开始日期
```

### 5.2 为什么用 MultiIndex？
- **方便截面操作**：`df.groupby(level=0).rank()` 按日期分组计算排名
- **LambdaRank 需要**：每个 Fold 的 group 信息（每天多少只股票）从 MultiIndex 的 level=0 提取
- **防数据错位**：(date, stock) 作为复合主键，确保唯一性

### 5.3 为什么最后输出转为列式？
- **存储友好**：Parquet 存储列式数据更高效
- **下游友好**：回测层读取时可以直接用 `df['date']` 筛选，不用处理 MultiIndex
- **去重方便**：`drop_duplicates(subset=['date', 'stock_code'])` 更直观

### 5.4 Fold 数量怎么确定？
```
Fold 数量 = (总交易日数 - min_required) / step

min_required = train_days + gap_train_valid + valid_days + gap_valid_test
             = 756 + 21 + 126 + 21 = 924 天

举例：2010-2024 共 3500 个交易日
可用长度 = 3500 - 924 = 2576 天
步长 = 63 天
Fold 数量 = 2576 / 63 ≈ 40 个

实际有 51 个 Fold，说明数据更长（约 4137 个交易日，16.4年）
```

## 八、常见问题

**Q1：为什么标签要用开盘价计算？**
> 因为 T 日收盘后才能拿到 T 日的因子值（如收盘后的技术指标），
> 实盘只能在 T+1 日开盘买入。用收盘价计算会有"看到未来"的偏差。

**Q2：为什么需要验证集？**
> LightGBM 用验证集做早停（Early Stopping），防止过拟合。
> 测试集用于最终评估，不能参与早停决策。

**Q3：Fold 之间为什么有重叠？**
> Fold 0 训练：2015-2018，Fold 1 训练：2015Q2-2018Q2
> 这是 Walk-forward 的特点：滚动窗口，用最近的历史预测最近的未来。

**Q4：实盘预测为什么只用最后一个 Fold 的模型？**
> 最后一个 Fold 用到了最新的数据训练，对当前市场状态适应最好。
