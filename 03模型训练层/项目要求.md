## 三、数据输入输出规范

### 3.1 输入数据来源

| 来源路径 | 内容 | 格式 | 用途 |
|---------|------|------|------|
| `02因子库/processed_data/factors/technical/` | 技术因子（8个） | Parquet宽表 | 特征(X) |
| `02因子库/processed_data/factors/financial/` | 财务因子（17个） | Parquet宽表 | 特征(X) |
| `02因子库/processed_data/market_data/close.parquet` | 收盘价 | Parquet宽表 | 计算标签(y) |

**⚠️ 重要提示**：行情数据目录 `market_data/` 下虽然有多个文件（open/high/low/close/volume/amount/suspendFlag），但**模型训练层只用 close.parquet** 来计算未来20日收益标签，其他行情数据不在训练层使用。

**因子清单**：
- 技术因子（8）：ret20, ret60, ret120, ret20_60, std20, std60, atr20, volatility_regime
- 财务因子（17）：pe, pb, ps, ey, roe, roa, roe_growth, opm, profit_growth, revenue_growth, oper_profit_growth, asset_turnover, financial_leverage, profit_quality, current_asset_ratio, current_ratio

### 3.2 输入数据格式

**⚠️ 重要：Parquet文件没有显式索引，`time` 是普通列而非索引！**

读取后需要手动设置索引：
```python
df = pd.read_parquet('ret20.parquet')
df = df.set_index('time')  # 必须手动设置索引
df.index = pd.to_datetime(df.index)
```

**宽表结构**（每个因子一个文件）：
```
文件: ret20.parquet

| time       | 000001.SZ | 000002.SZ | 000004.SZ | ... |
|------------|-----------|-----------|-----------|-----|
| 2010-01-04 | NaN       | NaN       | 0.05      | ... |  <- 未上市=NaN
| 2010-01-05 | NaN       | NaN       | 0.03      | ... |
| ...        | ...       | ...       | ...       | ... |
| 2026-02-27 | -0.02     | 0.01      | 0.04      | ... |
```

| 属性 | 规范 |
|------|------|
| 首列 | `time` (datetime64[ns])，**普通列而非索引** |
| 其余列 | 股票代码，如 `000001.SZ`，共约 5190 列 |
| 值 | float64，已清洗（去极值、中性化、标准化）|
| 缺失标记 | `None` 或 `NaN` |
| 日期范围 | 2010-01-04 至 2026-02-27 (共3920个交易日) |

### 3.3 数据对齐逻辑

**关键区分：两种"缺失"**

| 类型 | 说明 | 处理方式 |
|------|------|----------|
| **结构性缺失** | 股票当时未上市（如2010年宁德时代还未上市）| 正常，每期截面只取当天已上市股票 |
| **因子缺失** | 股票已上市，但因子值为NaN（如PE因亏损无法计算）| 该样本整行删除（dropna） |

**截面数据构造流程**（针对每个交易日）：
```python
for date in fold_dates:
    # 1. 取当天已上市股票（close不为NaN的）
    stocks_today = [s for s in all_stocks if close.loc[date, s] is not None]
    
    # 2. 取这些股票的25个因子值
    features = all_factors.loc[date, stocks_today]  # shape: (n_stocks, 25)
    
    # 3. 剔除因子值缺失的样本（任一因子为NaN则删除该股票）
    valid_mask = features.notna().all(axis=1)
    features_clean = features[valid_mask]
    
    # 4. 计算标签（未来20日收益）
    labels = close[t+20] / close[t] - 1
    labels = labels.reindex(features_clean.index)
    
    # 5. 标签也要有效（非NaN，即t+20未停牌）
    final_mask = labels.notna()
    X = features_clean[final_mask]   # 最终特征
    y = labels[final_mask]           # 最终标签
```

### 3.4 输出数据格式

**predictions.parquet**（历史回测预测，核心输出）：
| 列名 | 类型 | 说明 |

| `date` | datetime64[ns] | 预测日期（截面日）|
| `stock_code` | string | 股票代码 |
| `pred_score` | float64 | 模型预测的Alpha得分 |
| `actual_return` | float64 | 实际未来20日收益率（用于回测验证）|
| `fold_id` | int32 | 所属fold编号 |

**索引**：普通RangeIndex（**不是MultiIndex**，下游需要可自行转换）

**live_predictions.parquet**（实盘预测）：

格式与 `predictions.parquet` **完全一致**：
| 列名 | 类型 | 说明 |
|------|------|------|
| `date` | datetime64[ns] | 预测日期（最近20天）|
| `stock_code` | string | 股票代码 |

> **设计意图**：格式一致便于后续合并。在回测层可以将 `predictions.parquet` + `live_predictions.parquet` 合并，对最近的数据也进行回测验证（等未来20天后 actual_return 有值了再验证）。

**summary.parquet**（汇总报告）：
每个Fold的关键指标汇总，便于分析模型表现：

| `pred_std` | float | 预测分数标准差 |
| `ret_mean` | float | 实际收益均值 |
| `ret_std` | float | 实际收益标准差 |
| `atr20` ... | float | 各因子的重要性得分（25列）|

**feature_importance/fold_{id}.csv**：
| 列名 | 类型 | 说明 |

## 四、核心设计细节

### 4.1 标签构建 (Label)

| 配置项 | 值 | 说明 |
|--------|-----|------|
| **预测目标** | 未来20日收益率 | 连续值，回归问题 |
| **计算公式** | `label = (open[t+20] - open[t]) / open[t]` | 前复权开盘价 |
| **截面转换** | 否 | 直接用原始收益率，不转分位数 |
| **极端值处理** | 不处理 | 因子已清洗，标签不做额外处理 |

**注意**：构建标签时避免未来函数，使用t+1到t+20的收益率。

### 4.2 特征处理

| 配置项 | 值 | 说明 |
|--------|-----|------|
| **特征来源** | 因子库/processed_data/factors/ | 技术因子 + 财务因子 |
| **特征数量** | ~25个 | 根据实际存在的因子文件动态加载 |
| **NaN处理** | 删除样本 | 直接dropna()，不填充 |
| **股票池** | 全A股 | 不剔除ST/新股/停牌 |

**数据对齐逻辑**：
1. 读取所有因子宽表（index=日期, columns=股票代码）
2. 找到共同日期和共同股票
3. 横向合并所有因子 → 每个(date, stock)对应一个特征向量
4. 合并标签（未来20日收益）
5. 删除含NaN的样本

### 4.3 Walk-forward 滚动训练

**窗口配置**：
```python
WALK_FORWARD_CONFIG = {
    'train_window': '3Y',      # 训练窗口：3年
    'valid_window': '6M',      # 验证窗口：6个月（用于早停）
    'test_window': '3M',       # 测试窗口：3个月
    'step': '3M',              # 步长：每3个月滚动一次
}
```

**Fold概念**：
- Fold = 一次完整的训练+验证+测试周期
- 从2010年开始，每次往后挪3个月，直到数据结束
- 预计约 40-50 个 folds

**示例时间线**：
```
Fold 0:
  训练：2010-01 ~ 2012-12 (3年)
  验证：2013-01 ~ 2013-06 (6月)
  测试：2013-07 ~ 2013-09 (3月)

Fold 1:
  训练：2010-04 ~ 2013-03 (3年)
  验证：2013-04 ~ 2013-09 (6月)
  测试：2013-10 ~ 2013-12 (3月)

...以此类推
```

**防泄漏检查**：
- ✅ 特征只用t时刻及之前的数据
- ✅ 标签只用t+1到t+20的未来数据
- ✅ 训练/测试严格按时间分割
- ✅ 因子已在数据层完成中性化/标准化

### 4.4 LightGBM 模型配置

**基础参数**（经典A股配置）：
```python
LGB_PARAMS = {
    'objective': 'regression',
    'metric': 'mse',
    'boosting_type': 'gbdt',
    'num_leaves': 31,
    'learning_rate': 0.05,
    'feature_fraction': 0.8,     # 列采样
    'bagging_fraction': 0.8,     # 行采样
    'bagging_freq': 5,
    'verbose': -1,
    'num_threads': 8,
    'n_estimators': 500,         # 最大树数量
    'early_stopping_rounds': 50, # 早停轮数
}
```

**训练策略**：
- 每个Fold**从头训练**，不继承前一个模型
- 使用验证集做早停（early stopping）
- 不使用样本权重（市值/时间衰减）

### 4.6 双模型支持（Regression vs LambdaRank）

系统支持两种模型，通过 `config['model']['name']` 自动切换：

| 模型 | 配置文件 | 目标函数 | 输出含义 | 适用场景 |
|------|---------|---------|---------|---------|
| **LightGBM 回归** | `default_config.yaml` | `regression` (MSE) | 预期收益（可解释数值） | 需要预测具体收益值 |
| **LightGBM LambdaRank** | `rank_config.yaml` | `lambdarank` (NDCG) | 相关性分数（仅用于排序） | 解决预测值扎堆问题 |

#### 4.6.1 为什么需要 LambdaRank？

**问题：预测值扎堆（Prediction Clustering）**

在某些市场环境下（如2016年熔断），GBDT模型会对大量股票输出相同的预测值：
```
Fold 9 (2016年熔断期)：
- 2592只股票中，1592只预测值完全相同（61.4%）
- 导致 Alphalens 分位数分组失败（qcut 报错：Bin edges must be unique）
- IC 时间序列出现"断层"
```

**原因**：树模型在分布外（Out-of-Distribution）数据上，将大量样本分到同一个叶节点。

**LambdaRank 解决方案**：
- 直接优化股票排名（pairwise learning）
- 输出相关性分数而非收益预测
- 强制区分股票顺序，避免扎堆

#### 4.6.2 使用方式

```bash
# 标准回归模型（默认）
python main_train_v1.py --exp-id exp_001

# LambdaRank 排序模型
python main_train_v1.py --exp-id exp_rank --config configs/rank_config.yaml
```

#### 4.6.3 关键区别

**数据准备（Group 处理）**：
```python
# LambdaRank 需要 group 参数：每个交易日是一个 query
# group = [2592, 2593, 2594, ...]  # 每天的股票数量

train_data = lgb.Dataset(
    X_train.values,
    label=y_train.values,
    group=train_groups,  # 关键：传入 group
)
```

**预测值解释**：
```python
# 回归模型
pred_score = 0.05  # 预期涨 5%，可用于仓位分配

# LambdaRank
pred_score = 0.05  # 仅表示排名靠前，不能解释为收益
# 使用方式：
rank = df.groupby('date')['pred_score'].rank(ascending=False)
top50 = df[rank <= 50]  # 选前50名
```

#### 4.6.4 LambdaRank 配置要点

```yaml
model:
  name: "lightgbm_rank"
  params:
    objective: "lambdarank"    # 排序学习目标
    metric: "ndcg"             # 评估指标
    ndcg_eval_at: [10, 30, 50] # 评估 Top-K 排名质量
    
    # A股防过拟合（关键参数）
    max_depth: 5
    min_data_in_leaf: 150      # 每叶子最少150只股票
    learning_rate: 0.015       # 降低学习率
    n_estimators: 1500         # 增加树数量
    
    # 正则化
    lambda_l1: 0.5
    lambda_l2: 1.0
```

## 五、训练流程

### 5.1 主流程 (walk_forward_trainer.py)

```python
def run(self):
    splits = self.splitter.get_splits()  # 获取所有fold切分
    
    for fold_id, (train_dates, valid_dates, test_dates) in enumerate(splits):
        # 1. 构造数据（仅当前fold的日期）
        X_train, y_train = self.data_constructor.build(train_dates)
        X_valid, y_valid = self.data_constructor.build(valid_dates)
        X_test, y_test = self.data_constructor.build(test_dates)
        
        # 2. 训练
        model = LightGBMModel(self.config)
        model.fit(X_train, y_train, X_valid, y_valid)
        
        # 3. 预测
        predictions = model.predict(X_test)
        
        # 4. 保存结果
        self._save_fold_results(fold_id, model, predictions, 
                                test_dates, X_test.index)
    
    # 5. 合并所有fold预测
    self._aggregate_predictions()
    
    # 6. 生成汇总报告（自动）
    self._generate_summary_report()
    
    # 7. 生成可视化图表（自动）
    self._generate_visualizations()
    
    # 8. 生成实盘预测（最新无label数据）
    self._generate_live_predictions()
```

**内存策略**：
- 每次只构造当前Fold需要的数据，用完即释放
- 不一次性加载所有日期的特征矩阵（避免内存溢出）

### 5.2 输出文件保存

每个Fold训练完成后保存：
- `models/model_fold_{id}.pkl` - 模型文件
- `feature_importance/importance_fold_{id}.csv` - 特征重要性
- 预测结果缓存（最终合并为predictions.parquet）

训练全部完成后**自动**生成：
- `summary.parquet` - 各Fold指标汇总（人可读，便于分析）

- `ic_trend.png` - IC时间趋势图
- `feature_importance.png` - 特征重要性可视化
- `live_predictions.parquet` - 最新20天实盘预测

## 六、predictions.parquet 详解

### 6.1 predictions 是什么？

**predictions.parquet** 是模型训练层的**核心输出**，用于下游回测和评估。

**数据格式**：
| 列名 | 类型 | 说明 |
|------|------|------|
| `date` | datetime64[ns] | 预测日期（截面日）|
| `stock_code` | string | 股票代码 |
| `pred_score` | float64 | 模型预测的Alpha得分（未来20日收益的预测）|
| `actual_return` | float64 | 实际未来20日收益率（用于验证）|
| `fold_id` | int32 | 生成该预测的Fold编号 |

**索引**：普通RangeIndex（**注意：不是MultiIndex**，便于人眼查看和通用处理）

### 6.2 predictions 怎么来的？

**只来自「测试集」，无缝拼接而成！**

```
Fold 0: 训练(3年) → 验证(6月) → 测试(3个月: 2013-Q3)
                                      │
                                      ▼
                              生成 2013-Q3 的 predictions

Fold 1: 训练(3年) → 验证(6月) → 测试(3个月: 2013-Q4)
                                      │
                                      ▼
                              生成 2013-Q4 的 predictions

Fold 2: 训练(3年) → 验证(6月) → 测试(3个月: 2014-Q1)
                                      │
                                      ▼
                              生成 2014-Q1 的 predictions

... 以此类推
```

**关键设计**：
- `step = 3M`（步长）= `test_window = 3M`（测试窗口）
- 51个Fold的测试期**无缝拼接**，覆盖 2013-Q3 到 2026-Q1
- **无重叠、无遗漏**的完整预测序列

### 6.3 predictions 的用途

predictions.parquet 是 **下游回测层** 的主要输入：

```
predictions.parquet
        │
        ├─► Alphalens 分析 ──► IC/IR、分层收益（因子有效性评估）
        │
        └─► Backtrader 回测 ──► 严格交易回测（策略可行性验证）
```

**注意**：
- predictions 已经包含了 `pred_score`（预测分数）和 `actual_return`（实际收益）
- 不需要再重新计算标签，直接使用即可
- 下游使用时可根据需要 `set_index(['date', 'stock_code'])` 转为MultiIndex

## 七、live_predictions 详解（实盘预测）

### 7.1 为什么要 live_predictions？

**Walk-forward 的固有局限**：
- 要计算 T 日的label，需要 T+20 日的价格
- 数据最新到 2026-02-27，则最后 20 个交易日（约 2026-02-01 ~ 2026-02-27）无法计算label
- 这些日期**无法进入回测**，但**可以用于实盘**

**解决方案**：
- 用**最新Fold**训练好的模型
- 预测**最近20天**的数据（不需要label）
- 输出 `live_predictions.parquet` 供实盘交易使用

### 7.2 live_predictions 怎么来的？

```
数据时间线:
2010-01 ──────────────────────────────────────────────► 2026-02-27
                      │                               ▲
                      │                               │
               Walk-forward回测                       │
               (predictions.parquet)                 │
                                                    │
                                          最后20天无法回测
                                          但可实盘预测
                                                    │
            Fold 50 模型 ───────────────────────────┘
               (用最新模型预测最近数据)
               (live_predictions.parquet)
```

**生成流程**：
1. 找到最新的Fold（如 Fold 50）
2. 加载该Fold的模型 `models/model_fold_050.pkl`
3. 构造最近20天的特征数据（**不计算label**）
4. 模型预测 → 输出 `live_predictions.parquet`

### 7.3 live_predictions 的用途

- **实盘交易信号**：获取最新的选股预测
- **每日更新**：可单独运行脚本，用最新模型预测当日数据

```python
# 下游使用示例：合并历史回测和实盘预测，统一处理
import pandas as pd

pred = pd.read_parquet('predictions.parquet')
live_pred = pd.read_parquet('live_predictions.parquet')

# 合并（格式完全一致，可直接concat）
all_pred = pd.concat([pred, live_pred], ignore_index=True)

# 统一处理：如按日期选Top50股票
today_signals = all_pred[all_pred['date'] == '2026-02-27'].nlargest(50, 'pred_score')

# 注意：live_pred部分的actual_return目前为NaN，待20天后可回填验证
```
