# 月度数据更新方案

## 背景

原系统每次运行都全量下载所有数据，存在以下问题：

1. **前复权漂移**：前复权价格会随着分红除权而变化。例如某股票2024年分红，其2023年的前复权价在今天下载时会和去年下载时不同。
2. **财务修正**：公司可能发布业绩修正公告，同一报告期的数据会更新。
3. **效率问题**：全量下载耗时较长（约3-4小时），不需要每次都重跑。

## 更新策略

### 1. 行情数据（K线）- 全量覆盖

**策略**：每月从 `20100101` 重新下载所有行情数据，直接覆盖原文件。

**理由**：
- 前复权价格会随时间漂移，必须定期全量刷新修正
- 每月跑一次（周末），时间可接受（2-3小时）
- 新上市公司会自动包含

**数据范围**：2010年至今（约15年数据）

### 2. 财务数据 - 智能合并

**策略**：
1. 下载 `20230101` 以来的新数据
2. 与现有数据合并
3. 按 `['report_date', 'm_anntime']` 去重，保留最后出现的

**理由**：
- 2023年前的历史数据基本稳定，不需要重下
- 2023年以来的数据可能有修正，需要更新
- 保留所有不同公告日的版本（快报/正式报告/修正报告）

**去重逻辑详解**：

```python
combined = pd.concat([old_df, new_df])
combined = combined.drop_duplicates(
    subset=['report_date', 'm_anntime'], 
    keep='last'
)
```

**场景分析**：

| 场景 | 处理结果 |
|------|---------|
| 数据完全一样（report_date + m_anntime 都相同）| 新数据覆盖旧数据（无实质影响） |
| 公司发布修正公告（同一公告日更正数据）| 新数据替换旧数据（如净利润从100亿改为110亿） |
| 快报 vs 正式报告（report_date相同，m_anntime不同）| 保留两个版本（都保留） |

**关键字段说明**：
- `report_date`：报告期，如 `20240331` 表示2024年一季报
- `m_anntime`：公告日，如 `20240429` 表示2024年4月29日公告

### 3. 元数据 - 重新下载

**策略**：每次重新下载股票列表和行业映射。

**理由**：
- 数据量小，下载很快
- 确保包含最新的股票池和行业分类

## 使用方法

### 命令行

```bash
# 1. 月度更新（推荐每月周末运行）
python data_main.py --monthly

# 2. 全量下载（首次使用或需要完整刷新）
python data_main.py --full

# 3. 自定义财务数据起始日期
python data_main.py --monthly --financial-start 20240101

# 4. 查看帮助
python data_main.py --help
```

### Python代码

```python
from monthly_update import MonthlyDataUpdater

# 创建更新器
updater = MonthlyDataUpdater()

# 执行月度更新
updater.monthly_update(
    market_start='20100101',      # 行情起始日期
    financial_start='20230101'    # 财务起始日期
)
```

## 运行时间预估

| 数据类型 | 数据量 | 预估时间 |
|---------|-------|---------|
| 行情数据 | 5000+只股票 × 15年 | 2-3小时 |
| 财务数据 | 5000+只股票 × 2年 | 30-60分钟 |
| 元数据 | 股票列表 + 行业映射 | <5分钟 |
| **总计** | - | **3-4小时** |

**建议**：在周末非交易时段运行。

## 注意事项

1. **网络稳定**：下载过程需要保持网络连接，避免中断
2. **磁盘空间**：确保有足够空间（原始数据约5-10GB）
3. **QMT权限**：需要投研端VIP权限才能下载数据
4. **首次运行**：建议先使用 `--full` 全量下载，后续使用 `--monthly`

## 常见问题

**Q: 为什么行情数据要全量覆盖而不是增量？**

A: 因为前复权价格会随分红除权而变化。例如某股票2024年分红，其2023年的前复权价在今天下载时会和去年不同。必须全量刷新才能保证历史价格准确。

**Q: 财务数据为什么要保留2023年前的数据？**

A: 历史财务数据基本稳定，公司很少修正3年前的财报。而且早期财报已经没有快报/正式报告的区别，不需要更新。

**Q: 如果某只股票退市了会怎样？**

A: 月度更新会获取最新股票列表，退市股票不会被下载（旧文件保留，但不会再更新）。

**Q: 新股上市怎么处理？**

A: 月度更新会获取最新股票列表，包含新股。新股的历史数据会自动从 `20100101` 下载（但实际上只有上市后的数据）。
