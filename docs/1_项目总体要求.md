# 截面多因子量化选股系统 - 项目总体要求

---

## 一、项目愿景 (Project Vision)

### 1.1 核心目标

构建一个 **可复现、可维护、可快速迭代** 的截面多因子量化选股系统。

通过引入 AI 模型（LightGBM/XGBoost/Transformer）捕捉因子与收益率之间的**非线性关系**，实现超越基准指数的超额收益（Alpha）。

### 1.2 核心准则

> **逻辑领先于代码，相关性服务于因果。**

- 每个因子必须有清晰的因果逻辑（Causal Logic）
- 通过 15-20 个低相关因子流实现 Alpha 与 Beta 的分离
- 追求"交易圣杯"：跨周期的 Alpha 捕捉机器

## 二、系统架构 (System Architecture)

整个系统分为 **四个核心模块**，遵循 **"存算分离"** 和 **"低耦合"** 原则：

```
┌─────────────────────────────────────────────────────────────────┐
│                     截面多因子量化选股系统                        │
├─────────────────────────────────────────────────────────────────┤
│  Module 1        Module 2         Module 3        Module 4      │
│  ┌─────────┐    ┌─────────┐     ┌─────────┐    ┌─────────┐     │
│  │  Data   │───→│  Alpha  │────→│  Alpha  │───→│Optimizer│     │
│  │ Engine  │    │ Factory │     │Combiner │    │&Backtest│     │
│  │ 数据引擎 │    │ 因子工厂 │     │ AI训练  │    │优化回测 │     │
│  └─────────┘    └─────────┘     └─────────┘    └─────────┘     │
│       │              │               │              │           │
│       ▼              ▼               ▼              ▼           │
│   原始数据        因子计算         非线性建模      组合优化       │
│   PIT对齐        数据清洗         滚动训练        交易成本       │
└─────────────────────────────────────────────────────────────────┘
```

### 2.1 模块职责

| 模块 | 英文名 | 核心职责 | 关键技术 |
|------|--------|----------|----------|
| **模块1** | Data Engine | 原始数据搬运与存储 | Parquet格式、PIT对齐 |
| **模块2** | Alpha Factory | 因子计算与数据清洗 | 去极值、标准化、中性化 |
| **模块3** | Alpha Combiner | AI模型训练与集成 | LightGBM/XGBoost、Walk-forward |
| **模块4** | Optimizer & Backtest | 组合优化与回测 | 行业中性、换手率约束 |

### 2.2 模块详解

#### Module 1: Data Engine (数据引擎)

**职责**：原始行情（OHLCV）与财务数据（PIT格式）的搬运与存储

**关键设计**：
- **存储格式**：本地 Parquet（按股票代码索引），兼顾读取速度与压缩率
- **复权方式**：等比前复权（`front_ratio`），避免传统前复权的负数问题
- **财务数据**：采用 PIT（Point-in-Time）格式，杜绝未来函数

#### Module 2: Alpha Factory (因子工厂)

**职责**：特征工程核心，负责因子计算、数据清洗、中性化处理

**关键设计**：
- **因子分类**：按家族分类（技术因子、财务因子、机器学习因子）
- **因果逻辑**：每个因子必须包含经济学因果解释
- **PIT对齐**：确保财务数据在计算时已公告

#### Module 3: Alpha Combiner (AI 训练与集成)

**职责**：使用 AI 模型对多维度因子进行权重分配与非线性建模

**关键设计**：
- **模型选择**：LightGBM（处理非线性和缺失值）、XGBoost、Transformer
- **训练策略**：Walk-forward（滚动训练），防止过拟合
- **验证方式**：截面回归，预测下个月收益

#### Module 4: Optimizer & Backtest (组合优化与回测)

**职责**：将模型输出的预测分数转化为交易头寸

**关键设计**：
- **组合优化**：考虑行业中性、换手率约束
- **交易成本**：单边 1.5‰ 手续费 + 滑点模拟
- **风险指标**：最大回撤、夏普比率、胜率

## 三、目录结构规范

### 3.1 整体目录树

```
截面多因子模型/
├── configs/                    # 策略与模型参数配置 (YAML)
│
├── 数据/                        # 【Module 1: 数据引擎】
│   ├── data/raw_data/          # 原始数据 (行情+财务)
│   └── tests/                  # 数据检验脚本
│
├── 因子库/                      # 【Module 2: 因子工厂】
│   ├── src/
│   │   ├── data_engine/        # 数据对齐 (PIT+TTM)
│   │   ├── alpha_factory/      # 因子计算 (技术+财务)
│   │   └── processors/         # 因子清洗 (标准化)
│   └── processed_data/         # 处理后数据
│
├── models/                      # 【Module 3: AI训练】
│   ├── trainer.py              # 模型训练 (LightGBM/XGBoost)
│   ├── evaluator.py            # 模型评估 (IC/IR)
│   └── ensemble.py             # 模型集成
│
├── backtester/                  # 【Module 4: 回测优化】
│   ├── optimizer.py            # 组合优化
│   ├── engine.py               # 回测引擎
│   └── metrics.py              # 绩效分析
│
├── validation/                  # 数据检验统一入口
└── experiments/                 # 实验记录 (MLflow)
```

### 3.2 数据存储规范

**工业级标准：One Factor, One File**

每个因子存储为独立的 Parquet 宽表（行：日期，列：股票代码）：

```
processed_data/
├── market_data/              # 行情基础数据 (open/high/low/close...)
├── financial_data/           # 财务基础数据 (PIT对齐后的原始字段)
├── technical_factors/        # 技术因子 (ret20/rsi/macd...)
└── financial_factors/        # 财务因子 (pe/pb/roe...)
```

**文件格式规范**：
- **Index**: 时间 (Date)
- **Columns**: 股票代码 (Ticker)
- **Values**: 该股票在当天的因子值

## 四、各模块架构设计

### 4.1 Module 2: Alpha Factory (因子工厂) 架构

#### 设计原则
- **"一家族一文件"**：不采用"一因子一文件"，而是按因子家族（技术/财务/质量）聚合
- **因果逻辑优先**：每个因子必须包含经济学因果解释 Docstring
- **算子化设计**：原子操作可复用，便于遗传算法自动生成因子

#### 目录结构
```
alpha_factory/
├── technical/                    # 技术因子家族
│   ├── momentum.py               # MomentumFactors 类
│   │   ├── factor_ret20()        # 20日收益率
│   │   ├── factor_ret60()        # 60日收益率
│   │   └── factor_price_momentum() # 价格动量
│   ├── volatility.py             # 波动率家族
│   └── liquidity.py              # 流动性家族
│
├── financial/                    # 财务因子家族
│   ├── valuation.py              # ValuationFactors 类
│   │   ├── compute_pe()          # 市盈率
│   │   ├── compute_pb()          # 市净率
│   │   └── compute_ps()          # 市销率
│   ├── profitability.py          # 盈利家族 (ROE/ROA)
│   ├── quality.py                # 质量家族 (周转率/现金流)
│   └── growth.py                 # 成长家族 (YoY增长)
│
└── ml_factors/                   # 机器学习因子 (预留)
    └── embeddings.py             # 非线性特征嵌入
```

#### 因子计算流程
```
原始数据 (market_data/ financial_data/)
    │
    ▼
┌─────────────────┐
│  因子家族类      │  ← 加载所需数据
│  (XXXFactors)   │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│   因子计算方法   │  ← 因果逻辑计算
│  factor_xxx()   │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  输出因子宽表    │  → 保存到 processed_data/factors/
│  (因子值矩阵)    │
└─────────────────┘
```

### 4.2 Module 3: Alpha Combiner (AI训练) 架构

#### 设计目标
- **非线性建模**：捕捉因子与收益的非线性关系
- **防止过拟合**：Walk-forward 滚动训练
- **可解释性**：特征重要性分析

#### 核心组件
```
models/
├── trainer.py                    # 模型训练器
│   ├── prepare_data()            # 数据准备 (特征X, 标签y)
│   ├── walk_forward_split()      # 滚动划分训练/验证集
│   ├── train_lightgbm()          # LightGBM训练
│   └── train_xgboost()           # XGBoost训练
│
├── evaluator.py                  # 模型评估器
│   ├── calculate_ic()            # 信息系数计算
│   ├── calculate_ir()            # 信息比率计算
│   ├── plot_factor_importance()  # 特征重要性可视化
│   └── backtest_signal()         # 信号回测验证
│
└── ensemble.py                   # 模型集成 (预留)
    └── weighted_average()        # 多模型加权融合
```

#### Walk-forward 训练流程
```
时间轴:  2010    2015    2020    2024
         │       │       │       │
         ▼       ▼       ▼       ▼
Fold 1: [训练]──→[预测]
           2010-2014  →  2015
Fold 2:     [训练]──→[预测]
             2011-2015  →  2016
Fold 3:       [训练]──→[预测]
               2012-2016  →  2017
                ...
```

#### 标签生成规则
| 标签类型 | 计算方法 | 用途 |
|---------|---------|------|
| **收益率标签** | 未来20日收益率 | 回归模型 |
| **涨跌标签** | 未来20日涨跌 (0/1) | 分类模型 |
| **分位标签** | 截面分位排名 (1-10) | 排序模型 |

### 4.3 Module 4: Optimizer & Backtest (回测) 架构

#### 设计目标
- **组合优化**：将预测分数转化为实际可交易头寸
- **风险控制**：行业中性、换手率约束
- **成本模拟**：真实交易成本（手续费+滑点）

#### 核心组件
```
backtester/
├── optimizer.py                  # 组合优化器
│   ├── neutralize_industry()     # 行业中性约束
│   ├── constrain_turnover()      # 换手率约束
│   ├── optimize_weights()        # 最优化求解 (CVXPY)
│   └── generate_positions()      # 生成目标头寸
│
├── engine.py                     # 回测引擎
│   ├── initialize()              # 初始化参数
│   ├── handle_data()             # 每日处理逻辑
│   ├── execute_trades()          # 执行交易
│   └── calculate_pnl()           # 盈亏计算
│
└── metrics.py                    # 绩效分析
    ├── calculate_sharpe()        # 夏普比率
    ├── calculate_max_drawdown()  # 最大回撤
    ├── calculate_ir()            # 信息比率
    └── generate_report()         # 生成回测报告
```

#### 组合优化约束条件
| 约束类型 | 说明 | 数学表达 |
|---------|------|---------|
| **行业中性** | 各行业的权重偏离基准不超过±2% | \|w_i - w_benchmark_i\| ≤ 0.02 |
| **换手率约束** | 单日换手率不超过20% | \|w_t - w_{t-1}\|_1 ≤ 0.2 |
| **个股权重** | 单只股票权重不超过5% | 0 ≤ w_i ≤ 0.05 |
| **做空限制** | 不允许做空 | w_i ≥ 0 |

#### 回测流程
```
输入: 模型预测的因子分数 (每日截面)
    │
    ▼
┌─────────────────┐
│   组合优化       │  ← 考虑约束条件生成目标权重
│  (Optimizer)    │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│   交易执行       │  ← 模拟成交（考虑滑点、手续费）
│  (Execution)    │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│   盈亏计算       │  ← 每日盈亏、净值曲线
│    (PnL)        │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│   绩效归因       │  ← 收益分解、风险分析
│   (Analytics)   │
└─────────────────┘
```

## 五、数据标准 (Data Specification)

### 4.1 行情数据标准

| 属性 | 规范 |
|------|------|
| **频率** | 日频 (1d) |
| **复权** | 等比前复权 (`front_ratio`) |
| **列名** | time, open, high, low, close, volume, amount |
| **时间戳** | datetime.date (YYYY-MM-DD) |
| **存储** | 宽表格式（行：日期，列：股票） |

### 4.2 财务数据标准

| 属性 | 规范 |
|------|------|
| **格式** | PIT (Point-in-Time) |
| **对齐键** | m_anntime (公告日), report_date (报告期) |
| **处理** | TTM计算（流量表）、期末值（时点表） |
| **存储** | 宽表格式（PIT对齐到日频） |

### 4.3 元数据标准

| 属性 | 规范 |
|------|------|
| **行业分类** | 申万一级行业（31个行业） |
| **股票池** | 全A股 / 沪深300 / 中证500 / 中证1000 |
| **市值计算** | close × cap_stk（PIT对齐后） |

### 4.4 数据检验标准

**分层检验**：
1. **原始数据层**：完整性、API抽样回测
2. **处理后数据层**：PIT对齐检验、TTM计算检验、跨表一致性

**检验类型**：
- 完整性检验（缺失值比例）
- 一致性检验（日期范围、股票代码对齐）
- 业务规则检验（负数检查、异常值检测）

**报告格式**：Markdown 报告，包含通过/失败状态

## 七、关键术语表

| 术语 | 英文 | 解释 |
|------|------|------|
| **PIT对齐** | Point-in-Time | 按公告日将财务数据对齐到交易日历，避免未来函数 |
| **TTM** | Trailing Twelve Months | 滚动12个月，用于将累计财务数据转为年度可比数据 |
| **等比前复权** | Front Ratio Adjustment | 使用乘法公式调整历史价格，不会出现负数 |
| **中性化** | Neutralization | 通过回归剥离行业和市值Beta，保留纯Alpha |
| **MAD** | Median Absolute Deviation | 中位数绝对偏差，用于去极值 |
| **Walk-forward** | 滚动训练 | 用历史数据训练，预测未来数据，防止过拟合 |
| **IC** | Information Coefficient | 信息系数，因子与收益率的相关系数 |
| **IR** | Information Ratio | 信息比率，IC的均值/标准差 |
