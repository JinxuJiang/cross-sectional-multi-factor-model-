# 2.3 财务数据预处理

---

## 一、概述

**模块归属**: Module 1 - Data Engine (数据引擎) - 财务数据处理子模块

**处理目标**: 将原始季度财务数据转换为日频 PIT (Point-in-Time) 对齐的宽表格式

**核心挑战**:
- 财务数据是**季度**的，行情数据是**日频**的
- 需要使用**公告日**而非报告期来避免未来函数
- 流量表（利润表）需要计算 TTM，时点表（资产负债表）直接使用期末值

## 三、处理流程

### 3.1 整体流程图

```
原始财务数据 (季度)
    │
    ├──→ 清洗1: report_date 去重
    │        └── 同一报告期只保留第一次公告
    │
    ├──→ 清洗2: m_anntime 去重
    │        └── 同一公告日保留 report_date 最大的
    │
    ├──→ 字段分类处理
    │        ├── 流量表 (利润表) → 计算 TTM
    │        └── 时点表 (资产负债表) → 直接使用
    │
    ├──→ PIT 对齐
    │        └── 按 m_anntime 展开到日频
    │
    └──→ 输出宽表
             └── 行：日期，列：股票代码
```

## 四、两步清洗

### 4.1 清洗1: report_date 去重

**目的**: 同一报告期只保留第一次公告的数据

**逻辑**:
```python
# 按 (report_date, m_anntime) 排序，保留 first
df.sort_values(['report_date', 'm_anntime'])
df.drop_duplicates(subset=['report_date'], keep='first')
```

**原因**: 财报发布后可能有修正，只取第一次发布的原始数据

### 4.2 清洗2: m_anntime 去重

**目的**: 同一公告日只保留一个报告期的数据

**逻辑**:
```python
# 按 (m_anntime, report_date) 排序，保留 last（report_date 最大）
df.sort_values(['m_anntime', 'report_date'])
df.drop_duplicates(subset=['m_anntime'], keep='last')
```

**原因**: 同一天可能公告多个财报（如年报和一季报一起发），取最新的报告期

## 五、字段分类处理

### 5.1 流量表（利润表）- 需要 TTM

| 输出字段名 | 原始字段名 | 处理方式 |
|-----------|-----------|---------|
| `net_profit_ttm` | `net_profit_excl_min_int_inc` | 累计值→单季度→TTM |
| `revenue_ttm` | `revenue` | 累计值→单季度→TTM |
| `oper_profit_ttm` | `oper_profit` | 累计值→单季度→TTM |

**TTM 计算逻辑**:

```
Step 1: 累计值 → 单季度值
   - 同一年内: 单季度 = 本期累计 - 上期累计
   - 跨年 Q1 (3月份): 单季度 = 本期累计（累计值重新计数）

Step 2: 单季度值 → TTM
   - TTM = 当季 + 前3季（滚动4期求和）

示例:
   Q1累计: 100 → Q1单季度: 100
   Q2累计: 250 → Q2单季度: 250-100=150
   Q3累计: 450 → Q3单季度: 450-250=200
   Q4累计: 700 → Q4单季度: 700-450=250
   
   Q4 TTM = 100+150+200+250 = 700
```

### 5.2 时点表（资产负债表）- 直接使用期末值

| 输出字段名 | 原始字段名 | 处理方式 |
|-----------|-----------|---------|
| `cap_stk` | `cap_stk` | 直接使用 |
| `tot_assets` | `tot_assets` | 直接使用 |
| `tot_shrhldr_eqy` | `tot_shrhldr_eqy_excl_min_int` | 直接使用 |
| `total_current_assets` | `total_current_assets` | 直接使用 |

## 六、PIT 对齐（核心）

### 6.1 PIT 逻辑

**核心**: 根据公告日 `m_anntime` 将季度数据对齐到交易日历

**示例**:
```
财报数据: 2024Q1 于 2024-04-24 公告

对齐结果:
   2024-04-24 ~ 2024-08-19  →  使用 2024Q1 数据
   2024-08-20 ~ 2024-10-27  →  使用 2024Q2 数据（假设8/20公告）
   
关键: 在 2024-04-24 之前，2024Q1 数据不可用（避免未来函数）
```

### 6.2 对齐算法

```python
def align(financial_records, trading_calendar):
    """
    将财务记录对齐到交易日历
    
    Args:
        financial_records: [{'m_anntime': '20240424', 'value': 100}, ...]
        trading_calendar: [datetime.date(2024,1,1), ...]
    
    Returns:
        每个交易日对应的最新已公告财务数据
    """
    result = []
    records = sorted(financial_records, key=lambda x: x['m_anntime'])
    current_idx = 0
    
    for trade_date in trading_calendar:
        # 找到最新的已公告记录
        while (current_idx + 1 < len(records) and 
               trade_date >= records[current_idx + 1]['m_anntime']):
            current_idx += 1
        
        if trade_date >= records[current_idx]['m_anntime']:
            result.append(records[current_idx]['value'])
        else:
            result.append(np.nan)  # 公告前无数据
    
    return result
```

## 九、关键注意点

### 9.1 数据依赖

- **必须先有行情数据**: `market_data/close.parquet` 用于获取交易日历
- **内存管理**: 逐只股票处理，避免一次性加载所有数据

### 9.2 清洗顺序

必须在 TTM 计算前完成两步清洗，否则会导致:
- 重复计算（同一报告期多条记录）
- 数据错乱（同一天多个报告期）

### 9.3 Q1 检测

TTM 计算时判断跨年不能只看年份，要检查是否为 **3月份**（Q1特征）:
```python
is_q1 = (current_month == 3)
if is_q1 and current_year != prev_year:
    quarter_value = current_value  # 新年Q1，直接使用累计值
```
