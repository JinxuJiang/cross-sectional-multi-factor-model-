# 3.1 因子层总体要求

---

## 一、因子层定位

**模块归属**: Module 2 - Alpha Factory (因子工厂)

**核心职责**:
- 计算技术因子（动量、波动率、流动性等）
- 计算财务因子（估值、盈利、质量等）
- 实现"因果逻辑驱动"的因子挖掘

**设计原则**:
- **逻辑领先于代码**: 每个因子必须有清晰的经济学因果解释
- **一家族一文件**: 按因子家族分类，而非一因子一文件
- **算子化设计**: 原子操作可复用，支持遗传算法自动生成

---

## 二、核心设计准则

### 2.1 因果逻辑优先

> **"相关性服务于因果"**

每个因子必须包含 Docstring 说明其经济学逻辑：

```python
def factor_roe_growth(self):
    """
    ROE增长率因子
    
    因果逻辑:
    ---------
    ROE（净资产收益率）是衡量公司盈利能力的核心指标。
    ROE持续增长代表公司内生动力增强，且具备复利效应，
    在截面中应获得超额收益。
    
    计算方法:
    ---------
    ROE_Growth = ROE(t) - ROE(t-4)  # 同比去年同季度
    
    参考:
    -----
    - Fama-French 三因子模型中的盈利因子
    """
    return self.roe.diff(4)
```

### 2.2 一家族一文件

不采用"一因子一文件"，而是按**因子家族**聚合：

| 家族 | 文件 | 包含因子 |
|------|------|---------|
| **技术动量** | `momentum.py` | ret20, ret60, price_momentum |
| **技术波动** | `volatility.py` | std20, atr, rsi |
| **估值因子** | `valuation.py` | pe, pb, ps |
| **盈利因子** | `profitability.py` | roe, roa, roic |
| **质量因子** | `quality.py` | asset_turnover, cash_ratio |
| **成长因子** | `growth.py` | revenue_growth, profit_growth |

### 2.3 类设计规范

每个家族文件内部定义一个类，每个因子作为一个方法：

```python
class ValuationFactors:
    """估值因子家族"""
    
    def __init__(self, processed_data_path=None):
        self.processed_data_path = processed_data_path
        self._load_data()
    
    def _load_data(self):
        """加载所需数据"""
        self.close = load_parquet('market_data/close.parquet')
        self.cap_stk = load_parquet('financial_data/cap_stk.parquet')
        self.net_profit = load_parquet('financial_data/net_profit_ttm.parquet')
    
    def compute_pe(self):
        """计算PE（详细因果逻辑见方法Docstring）"""
        market_cap = self.close * self.cap_stk
        pe = market_cap / self.net_profit
        return pe
```

## 三、因子分类体系

### 3.1 技术因子 (Technical Factors)

基于价格和成交量的时序特征：

| 类别 | 代表因子 | 因果逻辑 |
|------|---------|---------|
| **动量** | ret20, ret60 | 市场参与者的行为偏差导致趋势延续 |
| **波动率** | std20, atr | 高波动通常伴随高风险，但短期反转 |
| **流动性** | turnover, amt_per_cap | 流动性溢价，低流动性股票收益更高 |
| **情绪** | rsi, bias | 极端情绪后的均值回归 |

### 3.2 财务因子 (Financial Factors)

基于财务报表的基本面特征：

| 类别 | 代表因子 | 因果逻辑 |
|------|---------|---------|
| **估值** | PE, PB, PS | 低估值股票长期具有价值回归效应 |
| **盈利** | ROE, ROA, ROIC | 高盈利公司具备持续竞争优势 |
| **质量** | accrual, cash_ratio | 高质量财务报表降低财务风险 |
| **成长** | profit_growth, revenue_growth | 成长性公司享有估值溢价 |

### 3.3 机器学习因子 (ML Factors) - 预留

非线性特征或嵌入向量：
- 自动编码器特征
- 图神经网络嵌入
- 时序模式识别

## 四、数据输入输出

### 4.1 输入数据

```
processed_data/
├── market_data/              # 行情数据
│   ├── close.parquet         # 收盘价
│   ├── volume.parquet        # 成交量
│   └── ...
│
└── financial_data/           # 财务数据（PIT对齐后）
    ├── net_profit_ttm.parquet
    ├── cap_stk.parquet
    └── ...
```

### 4.2 输出数据

```
processed_data/
├── technical_factors/        # 技术因子
│   ├── ret20.parquet
│   └── ...
│
└── financial_factors/        # 财务因子
    ├── pe_ratio.parquet
    └── ...
```

### 4.3 数据流

```
数据层输出 (processed_data/)
    │
    ├── market_data ──────┐
    │                      ▼
    ├── financial_data ──→ 因子家族类
    │                      (XXXFactors)
    │                      │
    │                      ▼
    │                   因子计算方法
    │                   (factor_xxx)
    │                      │
    │                      ▼
    └── industry.parquet  因子宽表输出
                          (processed_data/factors/)
```

## 六、关键术语

| 术语 | 解释 |
|------|------|
| **因子家族** | 按逻辑相关性分组的因子集合（如估值家族）|
| **因果逻辑** | 因子预测收益的经济学原理 |
| **算子化** | 将因子计算分解为可复用的原子操作 |
| **截面** | 某一时间点上所有股票的横截面数据 |
| **时序** | 单只股票的时间序列数据 |
