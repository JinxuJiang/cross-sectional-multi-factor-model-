## 二、关键决策记录

### 2.1 时间戳转换

| 决策项 | 内容 |
|--------|------|
| **原始格式** | 毫秒时间戳 (int64: 1262534400000) |
| **转换后格式** | datetime.date (2010-01-04) |
| **转换位置** | market_data_loader.py 中读取时转换 |
| **原因** | 便于阅读和理解，PIT对齐需要date对象 |

**代码片段**:
```python
raw_times = table.column('time').to_pylist()
times = [datetime.datetime.fromtimestamp(t / 1000).date() for t in raw_times]
```

### 2.2 财务字段映射（2026-02-26更新）

修正了原始数据字段名与输出字段名的映射关系：

| 输出字段 | 原始字段 | 说明 |
|---------|---------|------|
| `tot_shrhldr_eqy` | `tot_shrhldr_eqy_excl_min_int` | 归母权益（用于PB）|
| `net_profit` | `net_profit_excl_min_int_inc` | 归母净利润（用于PE）|
| ~~`roe_ttm`~~ | ~~删除~~ | 原始数据无此字段，后续从 net_profit_ttm / tot_shrhldr_eqy 计算 |

### 2.3 PIT对齐逻辑

```
财务数据（季度）:    2010Q1(4/29公告) ─────── 2010Q2(8/25公告)
                          │                         │
                          ▼                         ▼
PIT对齐后（日度）:  2010-04-29 ~ 2010-08-24  使用 Q1 数据
                    2010-08-25 ~ 2010-10-27  使用 Q2 数据
```

**交易日历**: 统一使用 `market_data/close.parquet` 中的日期

### 2.4 清洗顺序（4步法）

因子计算完成后，统一进入清洗流程：

```
1. 去极值 (MAD) ──→ 2. 填补缺失 (行业中位数) ──→ 3. 中性化 (OLS) ──→ 4. 标准化 (Z-Score)
```

**存储策略**: 清洗后的因子直接覆盖原文件（不保留原始因子）

### 2.5 中性化细节

| 中性化类型 | 实现方式 | 依赖数据 |
|-----------|---------|---------|
| **行业中性化** | OLS回归剥离行业Beta | industry_map.csv |
| **市值中性化** | OLS回归剥离市值Beta | close × cap_stk（需PIT对齐）|

**注意**: 行业数据来自静态文件，市值数据需要实时计算

### 2.6 复权方式决策（2026-03-02更新）

| 问题 | 传统前复权 (`front`) | 等比前复权 (`front_ratio`) |
|------|---------------------|---------------------------|
| **原理** | 加减法调整 | 乘法调整 |
| **问题** | 长期可能出现负数价格 | 不会出现负数 |
| **决策** | ❌ 不使用 | ✅ **采用** |

**改动**: `Base_DataEngine.py` 第22行，`dividend_type='front'` → `'front_ratio'`

## 六、注意事项

### 6.1 NaN处理

- **中性化时**: 无行业/无市值的股票直接 drop（剔除）
- **缺失值填补**: 按行业中位数或均值填补

### 6.2 存储规范

- **清洗后**: 直接覆盖原文件，不保留原始因子
- **字段命名**: 全英文小写，下划线连接，TTM加后缀
- **时间戳**: 统一使用 datetime.date 格式

### 6.3 内存优化

- **读取**: 使用 PyArrow 读取 parquet，避免 pandas 内存占用
- **处理**: 逐只股票处理，避免一次性加载所有数据
- **对齐**: 使用 numpy 向量化计算，避免循环

### 6.4 版本控制

- **数据**: 原始数据一旦下载不轻易删除
- **代码**: 关键决策记录在本文档
- **因子**: 清洗后直接覆盖，不保留历史版本（节省空间）
