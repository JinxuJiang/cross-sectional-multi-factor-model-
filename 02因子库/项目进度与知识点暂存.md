## 二、关键决策记录

### 1. Market Data 处理
- **时间戳转换**: 原始数据是毫秒时间戳(int64: 1262534400000)
- **转换后格式**: datetime.date (2010-01-04)
- **转换位置**: 在 market_data_loader.py 中读取时转换
- **文件格式**: 宽表（行：日期，列：股票代码）

### 2. 因子清洗流程 (Processors)
**处理顺序**:
1. 去极值 (MAD)
2. 填补缺失 (行业中位数 / 0)
3. 中性化 (OLS残差法)
4. 标准化 (Z-Score)

**存储策略**: 直接覆盖原文件（不保留原始因子）

### 3. 中性化细节
- **行业数据**: 来自 industry_map.csv（静态，不随时间变化）
- **市值数据**: 需要 close × cap_stk（需要PIT对齐）
- **行业中性化**: 可以先做（不依赖财务处理）
- **市值中性化**: 需要等财务数据处理完成

### 4. 财务数据处理
**命名规范**:
- 英文小写 + 下划线
- TTM字段加 `_ttm` 后缀
- 资产负债表字段无后缀

**处理字段（7个）**:
| 输出字段名 | 原始字段名 | 类型 | 说明 |
|-----------|-----------|------|------|
| cap_stk | cap_stk | 期末值 | 总股本，用于计算市值 |
| tot_assets | tot_assets | 期末值 | 总资产 |
| tot_shrhldr_eqy | tot_shrhldr_eqy_excl_min_int | 期末值 | 归属于母公司股东权益合计（用于PB） |
| total_current_assets | total_current_assets | 期末值 | 流动资产 |
| net_profit_ttm | net_profit_excl_min_int_inc | TTM | 归母净利润（用于PE、ROE） |
| revenue_ttm | revenue | TTM | 营业收入 |
| oper_profit_ttm | oper_profit | TTM | 营业利润 |

**字段映射说明**（2026-02-26更新）：
- 修正了原始数据字段名与输出字段名的映射关系
- `tot_shrhldr_eqy` → 原始字段 `tot_shrhldr_eqy_excl_min_int`（归母权益）
- `net_profit` → 原始字段 `net_profit_excl_min_int_inc`（归母净利润）
- 删除 `roe_ttm`（原始数据无此字段，后续从 net_profit_ttm / tot_shrhldr_eqy 计算）

**新增 `--fields` 参数支持**（2026-02-26更新）：
- `main_prepare_financial_data.py` 支持 `--fields` 参数指定处理特定字段
- 用法：`python main_prepare_financial_data.py --fields cap_stk tot_assets`
- 用于快速测试单个字段的处理速度

**PIT对齐逻辑**:
```
财务数据（季度）:    2010Q1(4/29公告) ─────── 2010Q2(8/25公告)
                          │                         │
                          ▼                         ▼
PIT对齐后（日度）:  2010-04-29 ~ 2010-08-24  使用 Q1 数据
                    2010-08-25 ~ 2010-10-27  使用 Q2 数据
```

**交易日历**: 使用 market_data/close.parquet 中的日期作为标准交易日历

## 六、注意事项

1. **NaN处理**: 中性化时无行业/无市值的股票直接 drop（剔除）
2. **存储**: 清洗后的因子直接覆盖，不保留原始
3. **字段命名**: 全英文小写，下划线连接，TTM加后缀
4. **交易日历**: 统一使用 close.parquet 的日期
5. **内存优化**: 使用 PyArrow 读取 parquet，避免 pandas
