# 财务数据处理流程文档

## 一、数据输入

**原始数据位置**: `数据/data/raw_data/financial_data/`

**格式**: 按股票分文件，如 `000001.SZ.parquet`

**关键字段**:
- `report_date`: 报告期（如 20240331 表示 2024年Q1）
- `m_anntime`: 公告日（如 20240424 表示 4月24日公告）
- 财务字段: net_profit_excl_min_int_inc, revenue, oper_profit, cap_stk 等

---

## 二、数据清洗（两步去重）

### 清洗1: report_date 去重
- **目的**: 同一报告期只保留第一次公告的数据
- **逻辑**: 按 `(report_date, m_anntime)` 排序，`keep='first'`
- **原因**: 财报发布后可能有修正，只取第一次发布的原始数据

### 清洗2: m_anntime 去重  
- **目的**: 同一公告日只保留一个报告期的数据
- **逻辑**: 按 `(m_anntime, report_date)` 排序，`keep='last'`（保留 report_date 最大的）
- **原因**: 同一天可能公告多个财报（如年报和一季报一起发），取最新的报告期

---

## 三、字段分类处理

### 流量表（利润表）- 需要 TTM

| 输出字段名 | 原始字段名 | 处理方式 |
|-----------|-----------|---------|
| net_profit_ttm | net_profit_excl_min_int_inc | 累计值→单季度→TTM |
| revenue_ttm | revenue | 累计值→单季度→TTM |
| oper_profit_ttm | oper_profit | 累计值→单季度→TTM |

**TTM 计算逻辑**:
1. 累计值 → 单季度值
   - 同一年内: `单季度 = 本期累计 - 上期累计`
   - 跨年Q1（3月份）: `单季度 = 本期累计`（累计值重新计数）
2. 单季度值 → TTM
   - 滚动4期求和: `TTM = 当季 + 前3季`

### 时点表（资产负债表）- 直接使用期末值

| 输出字段名 | 原始字段名 | 处理方式 |
|-----------|-----------|---------|
| cap_stk | cap_stk | 直接使用 |
| tot_assets | tot_assets | 直接使用 |
| tot_shrhldr_eqy | tot_shrhldr_eqy_excl_min_int | 直接使用 |
| total_current_assets | total_current_assets | 直接使用 |

---

## 四、PIT 对齐（Point-in-Time）

**核心逻辑**: 根据公告日 `m_anntime` 将季度数据对齐到交易日历

**示例**:
```
财报数据: 2024Q1 于 2024-04-24 公告
对齐结果: 从 2024-04-24 开始，每个交易日都使用 2024Q1 的数据
          直到 2024Q2 公告日（如 2024-08-20）前一日
```

---

## 五、输出格式

**输出位置**: `因子库/processed_data/financial_data/`

**文件格式**: 每个字段一个 parquet 文件

**文件结构**:
```
行: 交易日（从 market_data/close.parquet 获取交易日历）
列: 股票代码
值: 对应财务数据值
```

**输出文件清单**:
- `net_profit_ttm.parquet`
- `revenue_ttm.parquet`
- `oper_profit_ttm.parquet`
- `cap_stk.parquet`
- `tot_assets.parquet`
- `tot_shrhldr_eqy.parquet`
- `total_current_assets.parquet`

## 六、Python 运行流程

```

    ├── prepare_all_fields()

    │       │       │
    │       │       └── _process_single_stock()
    │       │               │
    │       │               ├── _read_financial_data()  
    │       │               │       └── 执行两步清洗
    │       │               │
    │       │               ├── 循环处理每个字段

    │       │               │       ├── 流量字段 (need_ttm=True):
    │       │               │       │       ├── calculate_ttm_from_cumulative()
    │       │               │       │       │       └── 计算 TTM
    │       │               │       │       └── align(value_field_ttm)
    │       │               │       │               └── PIT 对齐到日频

    │       │               │

    │       │

    │
    └── 保存 parquet 文件
```

---

## 七、关键注意点

1. **清洗顺序**: 必须在 TTM 计算前完成两步清洗
2. **Q1 检测**: TTM 计算时判断跨年不能只看年份，要检查是否为3月（Q1）
3. **数据依赖**: 需要先有 `market_data/close.parquet` 才能运行
4. **内存管理**: 逐只股票处理，避免一次性加载所有数据
